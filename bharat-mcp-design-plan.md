## Tool Output UX Guidelines

### Every tool response MUST include:
1. Source attribution: "Data from [source], retrieved [timestamp]"
2. _meta object: adapter, cached, fetchedAt, latencyMs, partial, locale

### State-specific copy:

**Not Found:** Always include next steps.
- "No GSTIN found for [input]. Verify the format: 15 chars, starting with state code (01-37)."
- "No company found matching [input]. Try searching by exact CIN or company name."

**Partial Data:** Label every missing field.
- "Data unavailable from [source]" on each missing field
- Top-level warning: "Some data sources are currently unavailable. Partial results shown."

**Error (API Down):** Be honest, offer alternatives.
- "GSTN is currently unavailable. Showing cached data from [date]. Some info may be outdated."

**Risk Score:** Natural language, not just numbers.
- "Risk Score: 7.2/10 (High Confidence) — 5/5 signals available."
- "Green: GST active, filings current."
- "Red: MCA annual return overdue 4 months."
- Low confidence: "Risk Score: 4.5/10 (Low Confidence) — 2/5 signals. Assess with caution."

**Hindi:** Conversational, not robotic.
- "यह GSTIN सक्रिय है।" (conversational) NOT "GSTIN स्थिति: सक्रिय" (robotic)
- "जोखिम स्कोर: 7.2/10 (उच्च विश्वसनीयता)"
