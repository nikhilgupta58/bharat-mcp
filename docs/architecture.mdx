## Module Breakdown

### GST Module

Handles all GST Intelligence tools. Connects to the GSTN Sandbox API (for development) or the production GSTN API.

| Tool | Description |
|---|---|
| `gstin_verify` | Real-time GSTIN status from GSTN |
| `hsn_search` | HSN/SAC code lookup with descriptions |
| `gst_rate_calculator` | Compute CGST/SGST/IGST on an amount |
| `gst_return_status` | Filing status for GSTR-1, GSTR-3B, etc. |
| `gstin_by_pan` | Discover GSTINs linked to a PAN |

### MCA Module

Connects to MCA21 public datasets and APIs.

| Tool | Description |
|---|---|
| `company_lookup` | Full company master by name or CIN |
| `director_search` | Directors and their associated companies |
| `cin_validate` | Validate CIN format and existence |
| `compliance_check` | Annual return and financial filing status |
| `director_network` | Graph of director вҶ” company relationships |

### Cross-Module Resolver

The resolver chains data across GST and MCA to produce unified entity profiles.

```
GSTIN вҶ’ extract PAN вҶ’ gstin_by_pan to confirm вҶ’
  PAN вҶ’ mca director_search вҶ’ company CINs вҶ’
    CINs вҶ’ company_lookup + compliance_check вҶ’
      all signals вҶ’ risk_score
```

## Language Layer

When `BHARAT_MCP_LANG=hi` is set, all tool outputs are passed through the i18n formatter before being returned to the MCP client. The formatter uses locale-aware number formatting (Indian numbering system) and Hindi field labels.
